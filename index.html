#Elements

MarkBook is designed to map easily onto elements commonly found in HTML, but also introduces several other elements.

#Elements inherited from HTML

The following elements are inherited from HTML:

p
Paragraph.
dl
Definition list. Can have a compact attribute (without value), which indicates a compact definition list.
dt
Definition list term.
dd
Definition list description.
ul
Unordered list.
ol
Ordered list.
li
List item.
code
Code.
em
Emphasis.
kbd
Keyboard input.
blockquote
Quotation, block form.

#Extra elements

The following elements are not part of HTML:

filename

A filename. This will typically be rendered in monospace, although occasionally italics are used to display fileames. In print, the word-breaking rules will generally be different for filenames, to avoid confusion caused by hyphenation marks. For example:

#p The file is written to %filename{output/about.html}.
firstterm

First occurrence of a term. Typically styled in italics. For example:

#p The %firstterm{kbd} element indicates keyboard input.
h

A header, typically found at the start of a document and at the beginnings of sections. See the description of the section element for an example.

link

A hyperlink. The target attribute contains the URL of the link target. For example:

#p For details, see %link[target=http://example.com/]{the example site}.
listing

Source code listing. The lang attribute contains the language of the source code. For example:

#listing[lang=ruby]
  def fact(n)
    if n > 1
      n * fact(n - 1)
    elsif n == 1
      1
    else
      raise ArgumentError, "Invalid argument: #{n}"
    end
  end
prompt

A prompt, as it appears in terminal output. For example:

#listing
  %prompt{$} %kbd{cd lib}
  %prompt{lib$} %kbd{pwd}
  /home/denis/nanoc/lib
  %prompt{lib$}

This could be rendered as follows:

$ cd lib
lib$ pwd
/home/denis/nanoc/lib
lib$
section

A section. This is intended to start with a header element (h). For example, the following code sample contains two nested sections:

#section %h{Use cases}
  #p General introduction to use cases goes here.

  #section %h{Use case 1: first term}
    #p Use case details go here.
